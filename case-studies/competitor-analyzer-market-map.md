# Competitor Analyzer: market-map decision

## Scenario

A team is considering a developer compliance product. It asks for a competitor analysis but has only a feature list from three vendors.

## Playbook output shape

The analysis first defines alternatives: existing vendors, an internal security spreadsheet/process, a consultancy, and doing nothing. It then requires dated source links and customer interviews before claiming pricing, demand, or product gaps. A proposed wedge must be tied to a buyer job—such as audit-evidence collection—not merely “more AI features.”

## Useful decision

Run five buyer interviews to test whether the bottleneck is evidence collection, remediation, procurement, or reporting. Do not build positioning around an unverified feature gap.
