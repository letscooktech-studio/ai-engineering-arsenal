# CTO Operating System evaluation rubric

| Dimension | A score of 4 requires |
| --- | --- |
| Outcome linkage | Connects engineering work to customer, business, or risk outcomes. |
| Signal quality | Defines metrics, dates, ownership, and important telemetry gaps. |
| Prioritization | Limits work, names trade-offs, and explicitly states what will not be done. |
| Operating cadence | Sets weekly/monthly/quarterly decisions, owners, escalation, and review points. |
| Calibration | Keeps people observations fair, aggregate, evidence-based, and appropriately uncertain. |
