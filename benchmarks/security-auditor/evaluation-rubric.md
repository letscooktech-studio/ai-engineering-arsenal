# Security Auditor evaluation rubric

Score 0–4 per dimension. Use the supplied artifact only; do not award points for invented findings.

| Dimension | A score of 4 requires |
| --- | --- |
| Coverage | Maps assets, trust boundary, authorization, input, secrets, dependencies, logging, availability, and supply-chain review scope. |
| Evidence | Every confirmed finding cites a supplied path, endpoint, or configuration and separates review gaps. |
| Exploitability | Explains safe attacker preconditions and impact without publishing a harmful exploit path. |
| Remediation | Gives a smallest durable fix, accountable owner type, and negative/regression test. |
| Calibration | Marks unknown runtime/configuration facts and does not overstate severity. |

See [failure patterns](failure-patterns.md).
